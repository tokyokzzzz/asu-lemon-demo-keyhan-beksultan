# TZ Analyzer - Система анализа технических заданий научных программ

Веб-приложение для анализа технических заданий (ТЗ) научно-технических программ Казахстана с использованием искусственного интеллекта на базе Google Gemini.

## Возможности

- 📄 Анализ документов в форматах .docx и .pdf
- 🤖 Автоматическое выявление проблем в техническом задании
- 📊 Оценка по 6 критериям (всего 100 баллов)
- 🔄 Применение исправлений и отслеживание истории версий
- 📥 Экспорт результатов в Excel
- 🌐 Поддержка русского и казахского языков
- 💾 Хранение истории анализов

## Требования к системе

- Docker и Docker Compose (для контейнеризации)
- Python 3.11+ (если запуск без Docker)
- Ключ API Google Generative AI (Gemini)

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd tz-analyzer
```

### 2. Подготовка файла конфигурации

Скопируйте файл примера конфигурации:

```bash
cp .env.example .env
```

### 3. Получение API ключа

1. Перейдите на https://aistudio.google.com/
2. Нажмите на "Get API Key" (или "Create API Key")
3. Выберите новый проект или существующий
4. Скопируйте ваш API ключ
5. Вставьте его в файл `.env`:

```
GEMINI_API_KEY=your_actual_api_key_here
```

### 4. Запуск приложения с Docker

```bash
docker-compose up --build
```

Приложение будет доступно по адресу: **http://localhost**

### 5. Остановка приложения

```bash
docker-compose down
```

## Ручная установка (без Docker)

### Установка зависимостей

```bash
cd backend
pip install -r requirements.txt
```

### Запуск backend сервера

```bash
cd backend
python main.py
```

Backend будет доступен на http://localhost:8000

### Запуск frontend

Откройте `frontend/index.html` в браузере или настройте веб-сервер для обслуживания файлов.

## Использование приложения

### Загрузка документа

1. Перейдите на вкладку "Анализ"
2. Перетащите файл ТЗ (.docx или .pdf) в зону загрузки или выберите его с помощью кнопки
3. Приложение автоматически проанализирует документ

### Просмотр результатов

После анализа вы увидите:

- **Оценка** (0-100 баллов) с визуальной шкалой
- **Разбор оценки** по 6 критериям:
  - Полнота разделов (25 баллов)
  - Ссылки на стратегические документы (20 баллов)
  - Количественные результаты (20 баллов)
  - Разбор бюджета (15 баллов)
  - Логическая согласованность (10 баллов)
  - Ясность языка (10 баллов)

- **Выявленные проблемы** с:
  - Описанием проблемы
  - Указанием серьезности (высокая/средняя/низкая)
  - Проблемным текстом
  - Предложенным исправлением

- **Рекомендации** для улучшения документа
- **Общая оценка** от AI

### Применение исправлений

1. Выберите проблемы, которые хотите исправить (отметьте чекбоксы)
2. Нажмите кнопку "Применить выбранные исправления"
3. Приложение создаст исправленную версию документа
4. Вы сможете:
   - Скачать исправленный DOCX файл
   - Просмотреть новую оценку и улучшение
   - Вернуться к предыдущей версии из истории

### История анализов

Перейдите на вкладку "История оценок" для просмотра:

- Всех проанализированных файлов
- Оценок до и после исправлений
- Процентов улучшения
- Дат анализа

Экспортируйте всю историю в Excel для отчетности.

## Критерии оценки

Нажмите на кнопку ℹ️ в зоне загрузки для просмотра полного описания всех критериев оценки.

### Основные требования к ТЗ:

1. **Обязательные разделы** - все 6 разделов должны быть заполнены
2. **Ссылки на стратегические документы** - должны быть конкретные пункты и параграфы, не просто названия
3. **Количественные показатели** - все результаты должны иметь числовые показатели с указанием параметров (квартили, патентные офисы и т.д.)
4. **Финансирование** - полная разбивка бюджета по годам
5. **Логическая согласованность** - задачи должны приводить к результатам
6. **Язык** - четкий, конкретный язык без расплывчатых формулировок

## API Endpoints

- **POST** `/api/analyze` - Загрузка и анализ документа
- **POST** `/api/apply-fix` - Применение исправлений
- **GET** `/api/download-excel` - Скачивание Excel с историей
- **GET** `/api/records` - Получение всех записей
- **GET** `/api/download/{filename}` - Скачивание файла
- **GET** `/api/health` - Проверка статуса API

## Трубка онке-шоу (Troubleshooting)

### Backend не запускается

Проверьте:
- Установлена ли переменная окружения `GEMINI_API_KEY`
- Доступен ли порт 8000
- Установлены ли все зависимости Python

Просмотрите логи:
```bash
docker-compose logs backend
```

### Frontend не загружается

Проверьте:
- Доступен ли Nginx на порту 80
- Нет ли ошибок в браузере (F12 → Console)
- Запущен ли backend

Просмотрите логи:
```bash
docker-compose logs nginx
```

### Анализ документа завершается с ошибкой

Причины:
- Документ повреждён или не читаем
- API ключ Gemini неверный или истёк
- Размер документа превышает лимит (10MB)

Проверьте логи:
```bash
docker-compose logs backend
```
